## 1. Documentación de Configuración (Días 54-57)

Esta sección proporciona una guía detallada de la configuración de todos los componentes de la infraestructura de QuickStay, permitiendo una fácil replicación, mantenimiento y resolución de problemas.

### 1.1. Manual de Instalación Paso a Paso

Se creará un manual exhaustivo que detalle los pasos para instalar y configurar cada servidor y servicio desde cero. Este manual incluirá:

*   **Preparación del Entorno:** Requisitos de hardware/software, configuración de red básica.
*   **Instalación del Sistema Operativo:** Pasos para instalar Ubuntu Server en cada VM.
*   **Configuración de Red:** Configuración de interfaces, VLANs, IP estáticas/DHCP.
*   **Instalación y Configuración de Servicios:** Pasos detallados para cada servicio (Apache, Tomcat, MySQL, AD-DC, Zabbix, Wazuh, Mosquitto, etc.).
*   **Configuración de Seguridad:** Aplicación de reglas de firewall (iptables), hardening básico.

### 1.2. Configuraciones de Dispositivos (Export)

Se exportarán y documentarán las configuraciones de todos los dispositivos y servicios clave. Esto facilitará la recuperación y la auditoría.

| Componente | Archivos de Configuración Clave | Ubicación (Ejemplo) |
| :--- | :--- | :--- |
| **Firewall (Linux)** | `/etc/network/interfaces`, `/etc/iptables/rules.v4` | Servidor Firewall |
| **Switch L3 (Linux)** | `/etc/network/interfaces` (VLANs, SVI) | Servidor Switch L3 (si virtualizado) |
| **Servidores Web (Apache/Nginx)** | `/etc/apache2/sites-available/quickstay.conf`, `/etc/nginx/sites-available/quickstay.conf` | Servidores Web |
| **Servidores de Aplicación (Tomcat)** | `/etc/tomcat9/server.xml`, `/var/lib/tomcat9/webapps/quickstay.xml` | Servidores de Aplicación |
| **Base de Datos (MySQL)** | `/etc/mysql/mysql.conf.d/mysqld.cnf` | Servidores de Base de Datos |
| **Active Directory (Samba AD-DC)** | `/etc/samba/smb.conf` | Controladores de Dominio |
| **Zabbix Server** | `/etc/zabbix/zabbix_server.conf`, `/etc/apache2/conf-available/zabbix.conf` | Servidor de Monitoreo |
| **Wazuh Manager** | `/var/ossec/etc/ossec.conf`, `/etc/filebeat/filebeat.yml` | Servidor de Seguridad |
| **Mosquitto MQTT** | `/etc/mosquitto/mosquitto.conf` | IoT Gateways |

### 1.3. Diagramas Actualizados (Visio/draw.io)

Se actualizarán y se incluirán los siguientes diagramas, preferiblemente creados con herramientas como Visio o draw.io, para una representación visual clara de la infraestructura:

*   **Diagrama de Topología Lógica:** Incluyendo el Servidor de Seguridad dedicado y las VLANs.
*   **Diagrama de Topología Física:** Representación de la ubicación física de los servidores y dispositivos de red.
*   **Diagrama de Flujo de Datos:** Cómo interactúan las diferentes capas de la aplicación y los servicios.
*   **Diagrama de Subnetting:** Detalle de las subredes y su asignación.

### 1.4. Scripts de Automatización (Si Aplica)

Se documentarán y se incluirán los scripts de automatización utilizados para la instalación, configuración o mantenimiento de la infraestructura (ej. scripts de hardening, scripts de backup, scripts de despliegue).

| Script | Descripción | Ubicación (Ejemplo) |
| :--- | :--- | :--- |
| `install_webserver.sh` | Script para automatizar la instalación y configuración de un servidor web. | `/opt/scripts/` |
| `backup_mysql.sh` | Script para realizar backups de la base de datos MySQL. | `/opt/scripts/` |
| `deploy_app.sh` | Script para desplegar la aplicación Java en los servidores de aplicación. | `/opt/scripts/` |

## 2. Documentación Operativa (Días 58-60)

Esta sección proporciona la información necesaria para la operación diaria, el mantenimiento y la gestión de incidentes de la infraestructura de QuickStay.

### 2.1. Manual de Administración Diaria

Una guía para las tareas rutinarias de administración del sistema, incluyendo:

*   **Monitorización:** Cómo revisar los dashboards de Zabbix y Wazuh, interpretar alertas.
*   **Gestión de Usuarios:** Procedimientos para crear, modificar y eliminar usuarios en Active Directory.
*   **Gestión de Logs:** Cómo acceder y analizar logs de diferentes servicios.
*   **Mantenimiento Básico:** Reinicio de servicios, verificación de espacio en disco.

### 2.2. Procedimientos de Backup/Restore

Documentación detallada de los procedimientos para realizar backups manuales y restaurar datos en diferentes escenarios.

*   **Backup Manual:** Pasos para iniciar un job de backup específico.
*   **Restauración de Archivos:** Procedimiento para recuperar archivos individuales.
*   **Restauración de VM:** Pasos para recuperar una máquina virtual completa.
*   **Restauración de Base de Datos:** Procedimiento para restaurar MySQL a un punto en el tiempo.

### 2.3. Plan de Mantenimiento Preventivo

Un calendario y una lista de tareas para el mantenimiento preventivo de la infraestructura.

| Tarea | Frecuencia | Responsable | Descripción |
| :--- | :--- | :--- | :--- |
| **Actualización de SO** | Mensual | Administrador de Sistemas | Aplicar parches de seguridad y actualizaciones del sistema operativo. |
| **Revisión de Logs** | Semanal | Administrador de Seguridad | Analizar logs de Zabbix y Wazuh en busca de anomalías. |
| **Verificación de Backups** | Semanal | Administrador de Sistemas | Confirmar la finalización exitosa de los jobs de backup. |
| **Limpieza de Discos** | Mensual | Administrador de Sistemas | Eliminar archivos temporales y logs antiguos. |

### 2.4. Matriz de Escalado de Incidencias

Un procedimiento claro para la gestión de incidentes, definiendo los niveles de soporte y los contactos.

| Nivel de Incidencia | Descripción | Primer Contacto | Escalado (si no se resuelve) |
| :--- | :--- | :--- | :--- |
| **Nivel 1 (Bajo)** | Problemas menores, preguntas de usuario. | Soporte Técnico (L1) | N/A |
| **Nivel 2 (Medio)** | Fallos de servicio no críticos, problemas de rendimiento. | Administrador de Sistemas | Administrador de Seguridad |
| **Nivel 3 (Alto)** | Fallo de servicio crítico, brecha de seguridad. | Administrador de Sistemas, Administrador de Seguridad | Gerencia de TI |
